ISO 27001 Risk Assessment Project.

Overview

This project demonstrates a structured information security risk assessment aligned with the principles of an Information Security Management System (ISMS) based on International Organization for Standardization ISO/IEC 27001.
The assessment was conducted for a simulated fintech organization, NovaPay Fintech Ltd, to identify critical assets, evaluate threats and vulnerabilities, assess risk levels and recommend appropriate security controls.
The objective is to demonstrate practical Governance, Risk and Compliance (GRC) capabilities used by security professionals to manage cybersecurity risks within organizations.

Project Objectives

This project demonstrates how organizations:

•	Identify critical information assets

•	Analyze cybersecurity threats and vulnerabilities

•	Evaluate risks using likelihood and impact analysis

•	Develop a structured risk register

•	Recommend risk treatment strategies

•	Support the implementation of an ISMS

Risk Assessment Methodology
The risk assessment followed a simplified methodology inspired by risk management practices within International Organization for Standardization ISO/IEC 27001.
The process included the following steps:
1.	Asset Identification
2.	Threat Identification
3.	Vulnerability Identification
4.	Risk Analysis and Evaluation
5.	Risk Treatment Planning
Each step contributes to understanding how potential security incidents could impact organizational operations.


Key Assets Identified

Examples of critical assets evaluated in this project include:

•	Customer Database

•	Payment Processing System

•	Cloud Infrastructure

•	Employee Workstations

•	Internal Network Infrastructure

These assets support NovaPay's financial services operations and require appropriate security protections.


Risk Evaluation Approach

Risks were evaluated using a qualitative risk analysis model based on:

•	Likelihood – probability of occurrence

•	Impact – potential organizational damage

The combination of these factors determines the overall risk level (Low, Medium, High, Critical).


Example Risks Identified

Some of the key risks identified during this assessment include:

•	Customer data breaches

•	Unauthorized access to payment systems

•	Malware infections on employee devices

•	Phishing attacks targeting employees

•	Cloud infrastructure misconfigurations

•	Ransomware attacks affecting organizational systems
Appropriate security controls were recommended to mitigate these risks.
